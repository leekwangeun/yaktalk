## 3.1 데이터 개요 (요약)

본 시스템은 식약처 공공데이터 4종을 수집·전처리하여 판정 근거로 사용하고, **동일한 공공데이터를 재료로 AI 학습 데이터를 직접 합성**한다. 즉 공공데이터가 (1) 위험도 판정의 근거이자 (2) AI 모델의 학습 재료라는 이중 역할을 한다.

| 구분 | 내용 | 규모 |
|---|---|---|
| 원천 공공데이터 | 허가정보·DUR품목·DUR성분·e약은요 (식약처) | 수집 20만 행 |
| 전처리 결과 DB | `drug_light.db` (제품명 사전·성분·DUR 규칙·M↔D 매핑) | 43.4MB |
| 합성 학습데이터 ① | 약 이름 인식(NER)·의도 분류용 질문 | 4만 문장 |
| 합성 학습데이터 ② | 응답 생성(KoGPT2)용 (검사결과→설명) 쌍 | 9천 쌍 |

## 3.2 약 이름 인식 모델 설계 — 규칙·통계·딥러닝 3단 앙상블

일반 사용자는 의학 용어가 아니라 상품명("타이레놀"), 통칭("감기약"), 오타("타이래놀")로 약을 표현한다. 이를 인식하기 위해 신뢰도가 높은 순서로 세 계층을 결합했다.

### (1) 1단계 — 사전 기반 최장일치 (Aho-Corasick)

허가정보에서 추출한 제품명 약 4.3만 개, 성분명, 카테고리어를 `pyahocorasick` 오토마타(트라이 + 실패 링크)로 구축한다. 입력 문장을 **한 번만 순회**하면서 사전에 등재된 모든 표현의 등장 위치를 동시에 탐지한다. 4.3만 개를 개별 대조하는 방식 대비 문장 길이에 비례하는 시간으로 처리되어 응답 지연이 없다. 겹치는 매칭은 최장일치를 취해 "판피린큐"가 "판피린"으로 잘못 좁혀지는 것을 방지한다.

### (2) 2단계 — 한글 자모 분해 유사도 (hgtk + rapidfuzz)

1단계에서 매칭되지 않은 어절(주로 오타)은 문자열 유사도로 보정한다. 이때 **음절(글자) 단위가 아니라 자모(초성·중성·종성) 단위**로 비교하는 것이 핵심이다. 한글 오타는 대부분 모음 하나 또는 받침 하나의 차이인데, 음절 단위 비교는 이 미세한 차이를 "한 글자 전체 불일치"로 과대평가한다.

`hgtk` 라이브러리로 글자를 자모로 분해한다:

```
레 = ㄹ + ㅔ + (받침 없음)
래 = ㄹ + ㅐ + (받침 없음)   → 중성 ㅔ↔ㅐ 하나만 차이
```

자모 분해 전후의 유사도 비교(실측):

| 입력(오타) → 정답 | 음절 단위 유사도 | 자모 단위 유사도 |
|---|---|---|
| 타이래놀 → 타이레놀 | 75.0 | **88.9** |
| 부루팬 → 부루펜 | 66.7 | **85.7** |
| 탁쎈 → 탁센 | 50.0 | **83.3** |

"탁쎈"은 음절 단위로는 2글자 중 1글자가 달라 50점(절반 불일치)이지만, 자모 단위(`ㅌㅏㄱㅆㅔㄴ` vs `ㅌㅏㄱㅅㅔㄴ`)로는 ㅆ↔ㅅ 하나만 달라 83.3점이다. 즉 자모 분해를 통해 **높은 임계값을 유지하면서도 오타를 포착**할 수 있어, 오탐(false positive)을 억제한다. 유사도 계산은 C++로 구현된 `rapidfuzz`를 사용해 대량 후보 비교도 밀리초 단위로 처리한다.

판정은 3구간 임계값으로 한다:

| 유사도 | 처리 | 예 |
|---|---|---|
| ≥ 85 | 자동 확정 (단, 첫 글자 일치 조건) | 타이래놀 → 타이레놀 |
| 75 ~ 85 | 되묻기("○○ 말씀이신가요?") | 애매한 축약어 |
| < 75 | 미인식 (추측 금지) | 오탐 위험 |

의료 도메인 특성상 애매하면 확정하지 않고 되묻는다. 실사용 평가 과정에서 다음 보정 규칙을 추가했다: 2글자 토큰은 임계값을 82로 상향(예: "임신"이 "이뮤신주"에 오매칭되는 것 차단), 영문·숫자는 정확 일치 강제(예: "비타민C"≠"비타민E"), 접두 일치 우선("판콜"→판콜에스·판콜아이 후보 제시).

### (3) 3단계 — KoELECTRA NER (직접 파인튜닝)

사전·자모가 포착하지 못하는 표현(신조어·축약)은 직접 파인튜닝한 KoELECTRA 개체명 인식(NER) 모델이 보완한다. 규칙(사전)이 1차, 딥러닝 모델이 백업인 **앙상블 구조**로, 모델이 틀려도 사전 매칭이 결과를 보장한다. NER이 서브워드 경계에서 이름을 잘라내는 경우 어절 경계까지 확장한 뒤 조사를 제거해 정규화한다.

### 설계 요약

| 계층 | 기법 | 라이브러리 | 역할 |
|---|---|---|---|
| 1 | 사전 최장일치 | pyahocorasick | 정확 표기 즉시 포착 |
| 2 | 자모 유사도 | hgtk + rapidfuzz | 오타·변형 보정 |
| 3 | NER 딥러닝 | KoELECTRA(직접 학습) | 사전 외 표현 보완 |

## 3.3 의도 분류 모델 설계 — KoELECTRA 문장 분류

같은 "병용 가능?"류 질문이라도 사용자의 의도는 다르다 — 함께 먹어도 되는지(병용), 부작용이 무엇인지(부작용), 어떻게 복용하는지(복용법), 그 외(기타). 의도에 따라 응답 로직이 갈리므로, 문장 전체를 4개 클래스로 분류하는 모델을 둔다.

### (1) 모델 구조

약 이름 인식과 동일한 KoELECTRA 베이스에 **문장 분류 헤드**를 얹어 파인튜닝한다. 입력 문장을 토큰화한 뒤 대표 토큰(`[CLS]`)의 표현으로 4클래스(기타·병용·복용법·부작용) 중 하나를 예측한다. 학습 데이터는 약 이름 인식과 동일한 합성 질문 4만 문장을 공유하며(각 문장이 NER 라벨과 의도 라벨을 함께 보유), 별도 데이터 제작 비용 없이 두 번째 모델을 학습한다.

### (2) 규칙 기반 폴백

의도 모델이 로드되지 않은 환경에서도 시스템이 동작하도록 키워드 규칙 폴백을 함께 구현했다("부작용/이상반응"→부작용, "몇 번/몇 알/식전/식후"→복용법, "같이/함께/병용"→병용, 그 외→기타). 이는 모델 대비 규칙의 정확도를 정량 비교하는 베이스라인 역할도 한다.

### (3) 판정 안정화 설계

의도 분류가 틀려도 안전하도록, **약이 2개 이상 인식되면 의도와 무관하게 병용 판정을 우선**한다. 실사용 평가에서 의도 분류 정확도는 95%였는데, 오분류 4건은 모두 약이 2개 이상인 경우여서 이 규칙에 의해 위험도 판정 결과에는 영향을 주지 않았다. 즉 분류 오류가 최종 안전성 판단으로 전파되지 않는 구조다.

## 3.4 응답 생성 모델 설계 — KoGPT2 + 사실 검증 게이트

응답 생성 역시 외부 API가 아니라 직접 파인튜닝한 **KoGPT2** 모델이 담당한다. 다만 의료 정보의 특성상 생성문을 그대로 신뢰하지 않고, 검증을 통과한 문장만 사용한다.

### (1) 학습 로직 — 프롬프트 마스킹

GPT 계열은 문장 전체에 대해 "다음 토큰 예측"을 학습하지만, 본 시스템은 **입력(검사 결과)은 학습에서 제외하고 출력(설명 문장)만 학습**한다. 입력 구간의 라벨을 `-100`으로 마스킹하면 손실 함수(CrossEntropyLoss)가 해당 구간을 무시하므로, 모델은 "검사 결과를 문장으로 변환하는 방법"만 학습하고 입력 자체를 암기하지 않는다.

```
학습 샘플:  [입력: A=… ; B=… ; 판정=주의 ; 근거=…] [출력: 두 약 모두 …]
라벨:       [-100, -100, …(입력 전부 무시)        ] [실제 토큰으로 손실 계산]
```

또한 KoGPT2는 토크나이저 특수 토큰을 명시하지 않으면 어휘 범위(51,200) 밖에 토큰이 추가되어 임베딩 오류가 발생하므로, `PreTrainedTokenizerFast`에 특수 토큰을 명시적으로 지정했다.

### (2) 추론 로직 — greedy 디코딩 + 4중 검증 게이트

- **greedy 디코딩**: 약 이름·수치를 정확히 복사하는 것이 핵심이므로 확률적 샘플링 대신 최고 확률 토큰만 선택한다(샘플링 시 "이트라코나졸"→"이트라코이나졸" 등 이름 변형 발생을 확인하고 비활성화).
- **사실 검증 게이트**: 생성문이 다음 검증 중 하나라도 실패하면 폐기하고 템플릿으로 폴백한다.

| # | 검증 | 차단 대상 |
|---|---|---|
| 1 | 숫자 검증 | 출력 수치는 입력에 있던 값이어야 함 (용량 지어내기) |
| 2 | 약 이름 검증 | 각 약 이름 앞 3글자가 출력에 등장 |
| 3 | 성분 검증 | 입력에 없는 성분명(DB 사전 대조) 등장 시 폐기 |
| 4 | 인용 검증 | 따옴표·괄호 안 문자열은 입력에 원문 그대로 존재 |
| 5 | 판정 일관성 | 등급(금기/정보없음)과 문장 논조 일치 |

이 구조로 "약 이해(KoELECTRA)와 응답 생성(KoGPT2) 모두 직접 학습한 모델이되, 의학적 판단은 100% 식약처 데이터가 담당하고, 생성문은 사실 검증을 통과해야만 사용자에게 전달"되는 안전 설계를 구현했다.

## 3.5 학습 데이터 합성 — 라벨 부재를 공공데이터로 해결

딥러닝 학습에는 "약 이름이 태깅된 한국어 문장"이 필요하나 공개 데이터가 존재하지 않는다. 이에 허가정보의 제품명 사전을 재료로 학습 데이터를 직접 생성했다.

### (1) NER·의도 데이터 (4만 문장)

**합성 방법 — 템플릿 × 슬롯 채우기.** 질문 템플릿 43종의 `{A}`, `{B}` 자리에 제품명(정규화된 2~8자 이름 21,225개 풀)을 대입한다. 카테고리어("감기약" 등)도 12% 확률로 혼합해 되묻기 상황을 학습시킨다.

세 가지 자동 처리로 데이터 품질을 확보한다:

1. **조사 자동 선택**: hgtk 받침 판별로 "테프란을/타이레놀을"(받침), "게보린을"이 아닌 상황별 조사를 정확히 부착. 문법 오류 문장 학습을 방지.
2. **오타 20% 주입**: 슬롯 채움 시 자모 단위 변형(모음 혼동·받침 탈락·음절 탈락)을 20% 확률로 주입. → 모델이 오타에 견고해짐.
3. **NER 라벨 자동 부착**: 슬롯 위치(문자 offset)를 알고 있으므로 라벨링이 자동. offset을 B-DRUG/I-DRUG/O 토큰 태그로 변환.

합성 예시:
```
문장:  "메가트루골드 먹기 전에 두통약 먹었는데 괜찮겠지?"
NER:   [0,6]=DRUG(메가트루골드), [13,16]=DRUG(두통약)
의도:  병용
```

데이터 구성: 총 4만 문장(train 36,000 / val 2,000 / test 2,000). 의도 4클래스 분포 — 병용 28,678 / 기타 4,524 / 부작용 3,777 / 복용법 3,021. 동일 문장이 NER 라벨과 의도 라벨을 함께 보유하여 하나의 데이터로 두 모델(인식·분류)을 학습한다.

**슬롯 개수와 다중 약물 인식의 관계.** 템플릿은 `{A}`, `{B}` 두 슬롯으로 구성되어 학습 문장에는 약이 최대 2개까지만 등장한다. 그러나 이는 3개 이상 약물의 인식을 제약하지 않는다. 약 이름 인식의 1차 계층은 학습 모델이 아니라 사전 스캔(Aho-Corasick)으로, 문장의 모든 위치를 순회하여 등재된 약을 **개수와 무관하게 전부** 탐지한다. 자모 유사도 보정 또한 각 어절을 독립적으로 처리하므로 세 번째·네 번째 약의 오타까지 개별 보정된다. NER 모델은 사전에 없는 표현의 백업이며, 토큰 단위 태거이므로 "약 이름의 패턴"을 학습할 뿐 등장 개수에는 독립적으로 일반화된다. 실측 결과 3~4개 약물(오타 혼합 포함) 입력이 모두 정상 인식되었으며, 인식된 약이 3개 이상이면 모든 쌍을 전수 비교(pairwise)하여 위험도를 판정한다.

### (2) 응답 생성 데이터 (9천 쌍)

**합성 방법 — (검사결과 직렬화 → 자연어 설명) 쌍.** 위험도 검사 결과를 구조화 문자열로 직렬화한 것을 입력으로, 대화체 설명을 출력으로 하는 쌍을 만든다. 시나리오 7종(정보없음·성분중복·효능군중복·병용금기·임부금기·노인주의·연령금기)마다 실제 DB의 제품명·금기 사유를 채우고, 같은 검사 결과에 말투 변형을 3~4개 부여해 표현 다양성을 확보한다.

합성 예시:
```
[입력] 입력: A=징크린주 ; B=없음 ; 판정=주의 ; 근거=임부금기:오말리주맙 (임부금기 2등급): …
[출력] 징크린주의 '오말리주맙'은 임부금기 2등급 성분이에요. 꼭 필요한 경우에만 의사 판단 하에 복용해야 해요.
```

**결정적 설계**: 입력 직렬화 함수를 런타임 생성기와 동일한 함수로 사용한다. 학습 시 본 입력 형식과 실전 입력이 문자 단위로 일치해야 생성 품질이 유지되기 때문이다.

데이터 구성: 총 9천 쌍(train 8,280 / val 360 / test 360). 학습에는 위 3.4(1)의 프롬프트 마스킹을 적용한다.

### 두 데이터셋 비교

| | NER·의도 (4만) | 응답 (9천) |
|---|---|---|
| 생성 방식 | 질문 템플릿 × 제품명 슬롯 | 검사결과 직렬화 × 설명 템플릿 |
| 재료 | 제품명 사전 21,225개 | DUR 규칙 + 제품명 |
| 라벨 | offset→BIO / 의도 4클래스 | 출력 문장 자체 |
| 특수 처리 | 조사 자동·오타 20% | 입력 형식=런타임 동일 |
| 학습 대상 | 약 이름 위치·질문 유형 | 데이터→대화체 변환 |

공통 원리는 **공공데이터(제품명·금기규칙)를 재료로 라벨링된 학습 데이터를 직접 생성**한 것이다.
